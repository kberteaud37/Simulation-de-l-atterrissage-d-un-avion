calcul_distance_aeroport

redirection_aeroport.calcul_distance_aeroport.calcul_distance_aeroport(code_depart, code_arrivee)[source]

Calcule la distance entre deux aéroports en milles nautiques (NM) en utilisant la formule de Haversine.

Cette fonction calcule la distance orthodromique (grand cercle) entre deux aéroports à partir de leurs coordonnées géographiques, en utilisant la formule mathématique de Haversine.

Paramètres:

  • code_depart (str) – Code OACI de l’aéroport de départ (ex: “CYUL”)

  • code_arrivee (str) – Code OACI de l’aéroport d’arrivée (ex: “KJFK”)

Renvoie:

Distance entre les deux aéroports en milles nautiques (NM)

Type renvoyé:

float

Lève:

  • IndexError – Si l’un des codes aéroport n’est pas trouvé dans la base de données

  • ValueError – Si les codes départ/arrivée sont identiques

Note:

  • 1 mille nautique (NM) = 1.852 km

  • Utilise le rayon terrestre moyen de 3440 NM (6,371 km)

  • La précision est d’environ ±0.3% en raison de l’hypothèse de sphéricité terrestre

compare

redirection_aeroport.compare.compare(monavion, piste, coef_secu=1.67)[source]

Compare la longueur des pistes avec la distance nécessaire à l’atterrissage et trouve une solution viable.

Cette fonction effectue une analyse de sécurité pour l’atterrissage en comparant :

  • La distance totale nécessaire (approche + transition + freinage) avec coefficient de sécurité

  • La longueur disponible des pistes

Si la piste initiale est trop courte, recherche automatiquement des alternatives :

  1. D’abord d’autres pistes du même aéroport

  2. Puis des aéroports voisins si nécessaire

Paramètres:

  • monavion (Avion) – Objet avion contenant les caractéristiques et méthodes de calcul

  • piste (Piste) – Piste d’atterrissage initialement prévue

  • coef_secu (float, optional) – Coefficient de sécurité (1.67 par défaut selon normes aéronautiques)

Renvoie:

Tuple contenant (piste sélectionnée, logs détaillés, distance nécessaire)

Type renvoyé:

tuple

Lève:

RuntimeError – Si aucun aéroport viable n’est trouvé après 20 recherches

Le tuple retourné contient :

  • piste (Piste): La piste sélectionnée (peut différer de la piste initiale)

  • logs (list): Liste de messages structurés (type, message) pour le suivi :

    • « info »: Informations techniques

    • « error »: Problèmes détectés

    • « success »: Solution trouvée

    • « text »: Messages de progression

  • distance_necessaire (float): Distance totale requise avec coefficient de sécurité

trouver_aeroport_proche

redirection_aeroport.trouver_aeroport_proche.trouver_aeroport_proche(exclusions)[source]

Trouve l’aéroport le plus proche en évitant les aéroports exclus.

Cette fonction utilise un arbre KD (K-Dimension) pour trouver efficacement l’aéroport le plus proche (distance orthodromique) qui n’est pas dans la liste d’exclusion.

Paramètres:

exclusions (list[Piste]) – Liste des aéroports à exclure de la recherche (doit contenir au moins l’aéroport de départ)

Renvoie:

DataFrame pandas contenant les informations de l’aéroport le plus proche

Type renvoyé:

pandas.Series

Lève:

ValueError –

  • Si l’aéroport de départ n’est pas trouvé dans la base de données

  • Si tous les aéroports valides sont exclus

  • Si la liste d’exclusion est vide

Note:

  • Utilise la formule de distance grand cercle via conversion en radians

  • La recherche est optimisée via KDTree pour des performances O(log n)

  • Les coordonnées sont converties en radians pour le calcul de distance
